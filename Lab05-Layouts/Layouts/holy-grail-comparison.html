🏆 Holy Grail Layout — 3 Methods Compared

Task 7 — Float vs Flexbox vs CSS Grid

🌊 Method 1: Float-Based
How it works: Sidebar floats left, aside floats right, main content sits between using margin-left and margin-right. Footer uses clear:both to drop below all floats. Clearfix needed on the inner wrapper.
Responsive: Remove floats and set width:100% at 768px.
Header
Aside

float:right 160px

Sidebar

float:left 160px

Main Content

margin-left:160px; margin-right:160px.
This is the classic float holy grail trick. Aside must come before main in HTML for float:right to work.

Footer — clear:both
↔️ Method 2: Flexbox
How it works: Outer container is display:flex; flex-direction:column. Inner body row is display:flex with sidebar, main (flex:1), aside. No margin tricks needed — flex handles spacing naturally.
Responsive: Change body row to flex-direction:column at 768px.
Header
Sidebar

width:160px
flex-shrink:0

Main Content

flex:1 takes all remaining space.
Cleaner than float — no margin hacks needed. HTML order matches visual order.

Aside

width:160px
flex-shrink:0

Footer
⊞ Method 3: CSS Grid
How it works: grid-template-areas defines the layout visually in CSS. Each child just declares grid-area: name. No float tricks, no flex direction changes — the most readable approach.
Responsive: Redefine grid-template-areas to single column at 768px.
Header
Sidebar

grid-area: sidebar
160px column

Main Content

grid-area: main — 1fr takes all remaining space.
Most readable of all three methods. Layout is defined entirely in CSS, not through HTML order or margin tricks.

Aside

grid-area: aside
160px column

Footer
